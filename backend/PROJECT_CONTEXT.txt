## 1. EVENT & THEME
- **Hackathon:** MIATHON'03
- **Theme:** "Thinking AI from Africa: ethics, power, and shared futures"
- **Team:** 4 Moroccan students
- **Sponsor alignment:** OCP (agriculture/water focus)

## 2. PROJECT IDENTITY
- **Name:** Filaha (فلاحة) — African Agricultural Intelligence
- **Concept:** Dual-sided ethical AI platform for African smallholders
  - **MAÏ** → Irrigation intelligence (when/how much to water)
  - **SILA** → Market intelligence (when to sell, at what price, direct buyer matching)
- **Tagline:** "الذكاء الزراعي الأفريقي" / "L'intelligence agricole africaine"
- **Scope:** Pan-African (Morocco = pilot/first lead)

## 3. TARGET USERS
| Side | User | Auth Method |
|------|------|-------------|
| B2C | African smallholder farmers | Phone number + OTP |
| B2B | Produce distributors/buyers | Email + password |

## 4. CRITICAL LANGUAGE RULE (DO NOT BREAK)
- **UI text visible to users:** Modern Standard Arabic (MSA / العربية الفصحى) by DEFAULT. French second. English third.
- **Code comments, variable names, function names, file names, folder names, SQL columns/tables, internal docs:** ENGLISH ONLY.
- **AI chat responses to farmers:** Formal Arabic (الفصحى), never Darija/dialect.
- **Developer-facing text:** English only.
- **i18n keys:** English snake_case (e.g., "irrigation.water_amount"), values in target language.
- **RTL:** Arabic UI must render right-to-left.

## 5. ETHICAL AI PILLARS (Theme Alignment)
Every AI feature must demonstrate:
1. **Explainability** — Farmer sees WHY the recommendation was given (factors, confidence, data sources)
2. **Transparency** — Farmer sees exactly which APIs/data were used
3. **Consent** — `data_sharing_consent` boolean; audit log of all AI decisions
4. **Data Ownership** — Farmers own their data; open-source crop models post-hackathon
5. **Shared Futures** — Cross-border community alerts (e.g., "Farmers in Tunisia face same pest")

## 7. MULTI-COUNTRY DATA ARCHITECTURE
Database is country-aware. Key tables:
- `countries` — ISO code, name in 3 languages, currency, data sources, active flag
- `regions` — climate zone, centroid, bounding box, linked to country
- `farms` — linked to country + region
- `price_history` — linked to country + data source (onca, wfp_vam, ratin, etc.)
- `crop_coefficients` — country-specific Kc values (FAO-56 based)
- `ai_explanations` — explainability log for every recommendation
- `data_audit_log` — ethics/transparency logging
